eMgine is the product of a course I have take at the University of Michigan; the course is EECS 498-007 Game Engine Architecture. This course was a deep dive into the inner workings of game engines and how they are built, and we made many iterations of building this game engine from building a text based game engine to a full 2D game engine with scripting from Lua with component based architecture.
This engine was built with the following tools: (Click item to visit cite!)
  • C++
  • Box2D
  • SDL2
  • GLM
  • Lua
  • jSON
  • LuaBridge
  • Dear ImGui
  • ImGui File Dialog
For my final project, I was tasked to build an extension for my game engine. I chose to build an editor that would allow users to create, edit, and run their games all in one location. I was inspired by other game engines like Unity and Unreal Engine. To the right is a video of the editor in action. My extension includes the building of scenes, actors, and components. The editor also has a text editor for editing lua files.
